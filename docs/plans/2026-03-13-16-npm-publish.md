# npm 首次发布 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Fix package.json export conditions, create initial changesets, and publish @dtax/tax-engine, @dtax/shared-types, @dtax/cli to npm registry.

**Architecture:** Use changesets for version management. Fix exports field ordering (types before conditional), create CLI tsup.config.ts for consistency, run dry-run publish to validate, then trigger actual publish via CI or manual `changeset publish`.

**Tech Stack:** changesets, tsup, pnpm workspaces, GitHub Actions

---

### Task 1: Fix Package Export Conditions

**Files:**

- Modify: `packages/tax-engine/package.json`
- Modify: `packages/shared-types/package.json`

**Step 1: Fix exports field — move "types" before conditional exports**

In `packages/tax-engine/package.json`, change the exports field to:

```json
{
  "exports": {
    ".": {
      "types": "./dist/index.d.ts",
      "import": "./dist/index.mjs",
      "require": "./dist/index.js"
    }
  }
}
```

Same for `packages/shared-types/package.json`.

If the `exports` field doesn't exist yet, add it. Ensure `main`, `module`, and `types` top-level fields remain for backward compatibility.

**Step 2: Verify build has no export warnings**

Run: `pnpm --filter @dtax/tax-engine build && pnpm --filter @dtax/shared-types build`
Expected: No "condition 'types' will never be used" warnings

**Step 3: Commit**

```bash
git add packages/tax-engine/package.json packages/shared-types/package.json
git commit -m "fix(packages): reorder exports field — types before conditional imports"
```

### Task 6: Publish to npm (Manual or CI)

**Option A: Manual publish (if NPM_TOKEN not yet in GitHub)**

**Step 1: Login to npm**

Run: `npm login` (or `npm login --scope=@dtax`)
Requires: npm account with `@dtax` org access

**Step 2: Publish**

Run: `pnpm publish-packages`
This runs: `pnpm build && changeset publish`

Expected:

```
@dtax/shared-types@0.2.0 published
@dtax/tax-engine@0.2.0 published
@dtax/cli@0.2.0 published
```

**Option B: CI publish (if NPM_TOKEN is configured)**

**Step 1: Set GitHub secret**

In GitHub repo settings → Secrets → Actions:

- Name: `NPM_TOKEN`
- Value: npm access token (from npmjs.com → Access Tokens → Generate New Token → Automation)

**Step 2: Push to main**

Run: `git push origin-private main`

CI will run `.github/workflows/publish.yml` which calls `changeset publish`.

**Step 3: Verify on npm**

Run:

```bash
npm view @dtax/tax-engine version
npm view @dtax/shared-types version
npm view @dtax/cli version
```

Expected: All show `0.2.0`
