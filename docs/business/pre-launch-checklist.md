### 0.3 路径 A：Phosmax → dTax B2B 服务模式（⭐ 首选，可立即开工）

#### 核心思路

你的 O1 授权你为 **Phosmax LLC** 工作。Phosmax 作为技术咨询公司，与 dTax 签订 **B2B Master Service Agreement (MSA)**，向 dTax 提供技术开发服务。你作为 Phosmax 的 W-2 员工执行这些服务。

```
你 (O1 持有人)
  ↓ W-2 雇员（已授权✅）
Phosmax LLC (S-Corp, O1 sponsor)
  ↓ B2B 服务合同
dTax Inc (Delaware C-Corp, 你持有股份)
```

**法律依据：**

- USCIS Policy Manual Vol. 2, Part M, Ch. 3 明确允许 O1 受益人自己的公司作为申请方
- 当 Company A 与 Company B 签订 B2B 服务协议时，Company A 的员工在法律上是**为 Company A 工作**，不是为 Company B
- 这与所有 IT 咨询/外包公司的运营模式完全一致
- **O1 比 H1B 灵活得多**：无 LCA（劳动条件申请），无工作地点限制，无 Neufeld Memo 式审查

#### 正确结构要素（缺一不可）

| 要素                           | 具体操作                                                                                          | 为什么重要                                  |
| ------------------------------ | ------------------------------------------------------------------------------------------------- | ------------------------------------------- |
| **1. 正式 MSA**                | Phosmax 与 dTax 签署 Master Service Agreement                                                     | 证明 B2B 关系，非雇佣关系                   |
| **2. SOW**                     | 每月/每季度的 Statement of Work（工作说明书）                                                     | 文档化服务内容                              |
| **3. Phosmax 开发票**          | Phosmax 按市场费率向 dTax 开月度发票                                                              | 证明 arm's-length 交易                      |
| **4. dTax 付款给 Phosmax**     | dTax → Phosmax（不是直接付给你）                                                                  | 资金流经公司，非个人                        |
| **5. 你仍是 Phosmax W-2 员工** | 工资从 Phosmax 发，Phosmax 发你 W-2                                                               | 维持已授权的雇佣关系                        |
| **6. dTax 董事会结构**         | 早期可以你担任 Sole Director（单一董事）；**配偶不算"独立"**；有可信顾问/朋友可邀其加入           | 独立性靠 MSA/SOW 文件证明，不强求独立董事   |
| **7. 你在 dTax 的角色**        | 可担任 Director、Incorporator、Shareholder，可持有 CEO/CTO 头衔；**但绝对不能从 dTax 直接领工资** | USCIS 认定"工作"的关键是工资/报酬，而非头衔 |
| **8. Phosmax 有其他业务**      | 保留或发展至少 1-2 个其他小客户                                                                   | 证明 Phosmax 不是 pass-through 空壳         |

#### MSA 关键条款

```
Master Service Agreement
Between: Phosmax LLC ("Provider") and dTax Inc ("Client")

服务范围: 软件开发、技术架构、CTO级技术咨询
计费方式: 按月固定费用 或 Time & Materials（按市场费率）
费率: $150-$250/小时（CTO级技术咨询市场价），或 $15,000-$25,000/月固定
IP 条款: 服务产出的 IP 归 dTax（标准 work-for-hire 条款）
付款: 月度发票，Net 30
期限: 12个月，自动续约
终止: 30天书面通知
```

> **费率定价很重要：** 必须是 arm's-length 市场费率。不能 100% pass-through（Phosmax 收 $X，转手全发给你做工资）。Phosmax 应保留合理利润（如10-20%），覆盖公司运营费用。

#### 关于"独立董事"的澄清

**配偶不能充当 O1 B2B 结构中的"独立董事"**，原因：

1. **不独立**：配偶与创始人有直接利益关联，USCIS 不会认可其作为独立治理证明
2. **角色混乱**：让配偶担任独立董事反而会加重 USCIS 对"两家公司实为一人控制"的质疑

**正确做法：**

- **早期（0-1年）：** 创始人担任 Sole Director（单一董事）完全正常，绝大多数 Stripe Atlas 注册的初创公司都如此
- **中期（融资前）：** 邀请一名你信任的业内朋友/顾问/天使投资人担任独立董事（需是美国公民/绿卡持有人最佳，但无硬性要求）
- **所谓"独立"的真正证明**：是 Phosmax ↔ dTax 之间**有纸质 MSA、月度 SOW、实际发票和银行转账记录**，而非必须有独立董事头衔

### 0.8 为什么 dTax 必须选 C-Corp

| 因素                | LLC                                        | C-Corp                                   |
| ------------------- | ------------------------------------------ | ---------------------------------------- |
| **O1 自雇 sponsor** | ⚠️ USCIS 审查更严，pass-through 收入像自雇 | ✅ 董事会结构清晰证明雇主-雇员关系       |
| **独立董事**        | 需要改 Operating Agreement 加独立 manager  | ✅ 标准董事会结构                        |
| **未来融资**        | 需 Delaware Flip 转 C-Corp ($2K-$5K)       | ✅ 已经是 C-Corp                         |

| **CA 首年免税**     | ❌ 已过期，首年即需 $800                   | ✅ C-Corp 首年免除 $800                  |

**结论：dTax 必须注册为 Delaware C-Corp。** 这不仅是为了融资和期权，更是为了 O1 签证合规。

## 一、公司注册（第1周）

### 1.1 结论先行：dTax 必须注册 Delaware C-Corp

基于你的 O1 签证身份（见第零节），dTax **必须注册为 Delaware C-Corp**，原因：

1. **O1 自雇 sponsor 要求：** C-Corp 董事会结构能清晰证明雇主-雇员关系

Atlas C-Corp 会**自动提交** Section 83(b) Election。如果自行注册 C-Corp，必须在收到股票后 **30天内** 向 IRS 邮寄 83(b) 选择表。这是**硬性截止日，无法补救**。错过意味着未来股票增值时需缴巨额税款。

### 1.3 加州外州公司注册（⚠️ 必须做，Stripe Atlas 不处理）

**你住在尔湾 (Irvine, CA) = 必须在加州注册为 Foreign Corporation。** 不能逃避。

加州对"在加州经营"的定义极其宽泛：创始人在加州家中工作即触发。不注册的后果包括：无法在加州法院起诉、每年 $2,000 罚款、补缴税金+利息、公司被暂停经营权。

**Stripe Atlas 只处理 Delaware 注册。** 加州注册必须你自己做（或用第三方服务）。Atlas 集成了 Mosey（州合规平台），但 Mosey 最低 $399/月 + $399实施费 + $300/州——对只注册一个州来说太贵。**DIY 仅需 ~$200，强烈推荐自己做。**

#### 关于 Delaware 地址

- Stripe Atlas 提供的 Delaware 地址 = **仅注册代理地址**，用于接收法律文件/政府通知
- **不能用作公司业务地址**
- IRS/银行/加州注册 → 使用你的**尔湾家庭地址**作为 principal office address
- 这完全正常且合法，绝大多数早期创业公司都用创始人家庭地址

- 3名辅助开发者，规模小，EOR 额外费用 $1,800-$2,100/月不合理

### 2.3 合规与 IP 保护措施

**付款流程：**

1. 每人签署英文+中文 **Independent Contractor Agreement**
2. 每人填写 **W-8BEN** 表格（外国人免税声明）
3. 通过 **Wise Business** 每月汇款（手续费 ~0.5%，2天到账）
4. 保留所有付款记录用于美国公司报税抵扣

**IP 保护（关键！）：**

⚠️ **研究发现：标准西方 NDA 在中国基本不可执行。** 必须使用 **NNN 协议**（Non-Use, Non-Disclosure, Non-Circumvention）。

1. **NNN 协议（替代 NDA）**— 必须满足以下条件：
   - **中文版本为准**（非英文）
   - **适用中国法律**（非 Delaware/美国法）
   - **约定中国法院管辖**（开发者所在地法院）
   - **包含违约金条款**（中国法院认可约定违约金）
   - **覆盖"不使用"**（开发者不能将代码用于自己的项目）
   - **预算：$2,000-$5,000 请中国 IP 律师起草**（Harris Sliwoski 等事务所）
2. **IP Assignment Agreement**（知识产权转让协议）— 所有代码归公司所有，中英双语
3. **代码管理**：GitHub Private Repo + 分支/模块级权限（每人只能访问自己负责的模块）
4. **数据隔离**：中国开发者**绝对不接触生产数据库**，使用脱敏/合成测试数据
5. **代码审查门控**：所有 PR 必须由创始人审核后合并
6. **可选：在中国注册软件著作权**（~$100-300，大幅增强执法地位）

**美国税务处理：**

- 付款计入 "Contract labor" 或 "Professional services"
- 年底填 Form 1099-NEC？**不需要** — 1099 仅适用于美国人，外国承包商用 W-8BEN
- 付款可全额作为业务费用抵扣

**出口管制：**

- 税务计算软件 = 商业软件，**不在 EAR 管控清单上**
- 不涉及加密技术（cryptography export）问题
- 不涉及 OFAC 制裁问题（中国不在全面制裁名单）

#### ✅ 推荐方案：Phantom Equity（幻影股权 / 现金增值权）

**Baker McKenzie（全球顶级律所）明确推荐：** "The usual practice for incentivizing Chinese employees at subsidiary level is to set up a Phantom Share Plan rather than a traditional stock option plan."

**核心优势：**

|                  | Phantom Equity        | 真实 NSO 期权         |
| ---------------- | --------------------- | --------------------- |
| 37号文 SAFE 登记 | ❌ 不需要             | ✅ 必须办理           |
| 美国 ITIN        | ❌ 不需要             | ✅ 需要               |
| 美国预扣税       | ❌ 不适用             | ✅ 30%（减免后约10%） |
| 流动性           | ✅ 触发事件时直接现金 | ❌ 私有阶段无法变现   |
| 合规总成本       | **$3K-$8K**           | $12K-$25K+            |
| 实施时间         | 2-4周                 | 3-6个月               |

#### Phantom Equity 运作机制

```
授予 Phantom Units（无实际股权转让）
    ↓
Vesting 期间（4年 + 1年 Cliff）
    ↓
触发事件：融资 / 收购 / IPO
    ↓
现金支付 = 公司估值 × 授予比例
（Mercury → Wise → 人民币银行账户）
```

**两种类型（选其一）：**

- **Full Value（全价值型）**：支付授予比例对应的公司总价值（更像真实股权）
- **Appreciation Only（增值型 / SAR）**：只支付"授予后"估值增长部分（激励效果更集中）

#### 给多少合适？（针对辅助开发者，非联合创始人）

| 角色                          | 建议 Phantom Equity | 说明                      |
| ----------------------------- | ------------------- | ------------------------- |
| 核心全职开发（联合创始人级）  | 1.0% - 2.0%         | 如果全职投入、长期绑定    |
| 重要兼职辅助开发（10-20h/周） | **0.2% - 0.5%**     | 你的3人场景               |
| 偶尔贡献的顾问                | 0.05% - 0.1%        |                           |
| **3人合计建议**               | **0.5% - 1.2%**     | 从10%期权池划出，影响极小 |

> ⚠️ 原文"每人2-5%，3人总计6-15%"是**联合创始人级别**的比例，辅助开发者给这么多会严重稀释未来融资空间，且产生巨额现金支付义务。

| 条款           | 推荐设置                                                          |
| -------------- | ----------------------------------------------------------------- |
| **授予比例**   | 每人 0.2%-0.5%，3人总计 0.5%-1.2%                                 |
| **Vesting**    | 4年 + 1年 cliff（标准）                                           |
| **估值方法**   | 最近一轮融资估值；无融资时用"年收入 × 5-10倍"公式                 |
| **支付触发**   | Change of Control / IPO（优先）；可加：年度分红触发（现金流好时） |
| **支付方式**   | 现金，Mercury → Wise → 人民币                                     |
| **没收条款**   | 违反 NNN / 竞业 → 没收全部未支付部分                              |
| **非转让**     | 不可转让                                                          |
| **无股东权利** | 无投票权、无信息权、无优先购买权                                  |

#### ⚠️ 409A 合规（重要）

Phantom Equity 属于 **Nonqualified Deferred Compensation**，受 Section 409A 约束：

- 支付时间必须在协议中**事先确定**（不能事后随意更改）
- 不能赋予持有人自行选择支付时间的权利
- 违规后果：持有人面临**即时征税 + 20% 附加税 + 利息**
- **务必让律师写入 409A safe harbor 条款**

#### 中国端税务

| 时间点       | 征税                                                            |
| ------------ | --------------------------------------------------------------- |
| 授予时       | ❌ 不征税                                                       |
| Vesting 期间 | ❌ 不征税（无实际收入）                                         |
| 收到现金时   | ✅ 按劳务报酬所得征税（20%-40%累进；或视认定为工资薪金 3%-45%） |

**支付凭证**：合同注明为"绩效激励 / Performance Bonus"，对应独立承包商协议，中国端自行完税。

#### 未来升级路径

```
现在（C-Corp + Phantom Equity）
    │
    ├── 融资（Pre-Seed / Seed）
    │   └── Phantom Equity 可继续保留，或谈判转换为 NSO（届时中国开发者
    │       可同步办理 37号文 SAFE 登记，有律所协助相对更容易）

#### 常见错误（必须避免）

1. ❌ 直接发真实股票给中国居民 → 无 37号文登记 = 无法合法变现，且违反外汇管制
2. ❌ Phantom Equity 比例设太高（每人2-5%）→ 未来须支付巨额现金，严重稀释
3. ❌ 口头承诺"以后给股权" → 中国法律不认可口头承诺，无法执行
4. ❌ Atlas 注册时未预留期权池 → 后续增加期权池需要股东会决议 + 重新签文件
5. ❌ 忽略 409A 合规 → 中国开发者收到现金时面临额外 20% 美国罚税
6. ❌ 触发事件只写 IPO → 对辅助开发者激励效果为零（IPO 遥不可及）
   └── 按合同约定比例现金结算

```

## 四、数据合规要求

### 4.1 需要什么？不需要什么？

| 要求 | 是否需要 | 时间点 |
|------|---------|--------|
| **SOC 2** | ❌ 上线不需要 | 有企业客户后考虑 (Year 2+) |
| **GDPR** | ✅ 必须（有EU用户） | 上线前：Privacy Policy + Cookie Consent + 数据删除功能 |
| **CCPA** | ✅ 建议（加州用户） | 上线前：Privacy Policy 中包含 CCPA 条款 |
| **E&O 保险** | ❌ 上线不需要 | 有CPA客户后考虑 ($500-$2K/年) |
| **PCI DSS** | ❌ | Stripe 处理支付，不需自行处理卡号 |

### 4.2 GDPR 最低合规（必做）

1. **Privacy Policy** 包含：数据收集范围、存储位置（美国）、第三方共享（Stripe/Resend/PostHog）、用户权利（访问/更正/删除/可移植性）、法律依据、保留期限
2. **Cookie Consent Banner**（CookieScript 免费版）
3. **数据删除 API**（需确认 DELETE /auth/account 端点是否完整删除所有用户数据）
4. **数据导出功能**（用户可导出自己的交易和报告数据）
5. **CCPA "Do Not Sell" 链接**（即使不卖数据也需要）
6. **IRS Circular 230 免责声明**（税务内容不能被用于避免 IRS 处罚）

### 4.3 运营州注意事项

**关键：如果你在非 Delaware 州实际运营（如加州），需要：**
- 在该州注册为 Foreign LLC（各州 $50-$500）
- 加州额外 $800/年最低 franchise tax（谨慎选择运营州）
- 8个新州在 2025 年生效了隐私法（Delaware、Iowa、NH、NJ、TN、MN、MD、KY）
