# DTax 完整网站规划 — 多角色深度分析

> **目标：** 将 DTax 从"工具型 SPA"升级为"完整的加密税务产品平台"，覆盖营销转化、用户引导、多角色支持、法律合规全链路。

---

## 第一部分：角色分析

### 角色 1：免费用户（个人散户）

**画像：** 持有 BTC/ETH 等主流资产，年交易 < 50 笔，第一次报加密税。

**当前痛点：**

- 注册后看到空 Dashboard，不知道下一步做什么
- 不了解 FIFO/LIFO 的区别，不知道选哪个
- 没有引导帮助理解 Form 8949 是什么
- 担心数据安全（"我的交易记录会被共享吗？"）

**需要的页面/功能：**

1. **新用户引导向导**（3-5 步）：选交易所 → 选导入方式 → 导入 → 看到结果
2. **功能介绍页** — 用非技术语言解释每个功能
3. **FAQ 页面** — "什么是 FIFO？""我需要报税吗？"
4. **隐私政策页** — 明确"数据不上传到第三方"
5. **税务术语词汇表** — 帮助新手理解

**关键指标：** 注册→首次导入 < 3 分钟，导入→看到税务摘要 < 1 分钟

---

### 角色 2：付费用户（活跃交易者 / DeFi 用户）

**画像：** 年交易 500-10,000 笔，使用多个交易所和 DeFi 协议，需要准确的税务报告。

**当前痛点：**

- 多交易所数据合并流程不直观
- 不知道 DTax 是否覆盖自己用的所有交易所
- 想要 Wash Sale 和 Tax-Loss Harvesting 优化
- 需要导出给会计师或 TurboTax 的报告格式

**需要的页面/功能：**

1. **交易所集成页** — 每个交易所的导入指南（"如何从 Binance 导出 CSV"）
2. **功能对比页** — 免费 vs Pro 的详细差异
3. **独立定价页** — 更详细的方案说明，附带 FAQ
4. **TurboTax/H&R Block 导入指南** — 如何使用 DTax 的输出
5. **升级引导** — 到达 50 笔限制时的平滑升级体验

**关键指标：** Free→Pro 转化率 > 5%，年留存率 > 60%

---

### 角色 3：税务专业人士 / CPA

**画像：** 管理 20-200 个客户，每年报税季集中处理，需要批量操作和专业报告。

**当前痛点：**

- 没有多客户管理界面
- 没有批量导入/导出能力
- 没有白标报告（给客户的报告上显示的是 DTax 品牌）
- 不了解 DTax 是否符合 IRS 标准

**需要的页面/功能：**

1. **专业版介绍页** — 面向 CPA 的独立 Landing Page
2. **合规声明页** — DTax 的计算方法符合 IRS Publication 544/550
3. **多客户管理功能**（产品功能，非页面）
4. **案例研究** — "XX 会计事务所如何用 DTax 处理 Y 个客户"
5. **Demo 预约** — 联系销售的入口

**关键指标：** CPA 注册→首个客户报告 < 30 分钟，席位扩展率

---

### 角色 4：企业用户（加密基金 / 交易所合规部门）

**画像：** 需要私有部署、SLA、API 集成、审计日志。

**当前痛点：**

- 不确定 AGPL 许可是否允许内部使用
- 需要了解私有部署的技术要求
- 需要 API 文档（已有 Swagger，但缺少企业级说明）
- 需要安全评估报告

**需要的页面/功能：**

1. **企业版页面** — 私有部署 + API + SLA 说明
2. **安全白皮书/页面** — 加密方式、数据隔离、审计日志
3. **自托管部署指南** — 比 README 更详细的文档
4. **许可证说明页** — AGPL 对企业意味着什么
5. **联系销售表单** — 企业咨询入口

**关键指标：** 企业咨询→签约转化率，部署时间

---

### 角色 5：超级管理员（DTax 平台运营）

**画像：** DTax SaaS 平台的管理员，管理用户、监控系统、处理支持请求。

**当前痛点：**

- 已有基础 Admin API（用户列表、角色管理），但无 Admin UI
- 没有用户增长、收入等业务仪表盘
- 没有内容管理系统（博客、公告）
- 没有用户支持工单系统

**需要的页面/功能：**

1. **Admin Dashboard**（产品功能）— 用户统计、收入、系统健康
2. **用户管理 UI** — 搜索/查看/禁用用户
3. **公告系统** — 新功能、维护通知
4. **不在 MVP 范围** — CMS、工单系统（可用第三方如 Crisp/Intercom）

**关键指标：** 问题响应时间，系统可用性

### 角色 6：运维专员 / 开发者

**画像：** 负责部署、监控、排障的技术人员，或想自托管的开源社区开发者。

**当前痛点：**

- Docker 部署文档在 README，不够详细
- 没有环境变量完整说明
- 没有故障排查指南
- 没有 API 变更日志

**需要的页面/功能：**

1. **文档站点** — 部署、配置、API、故障排查
2. **Changelog 页面** — 版本更新记录
3. **贡献指南** — 开源社区贡献流程
4. **状态页** — 系统运行状态（可用第三方如 Upptime）

**关键指标：** 自托管部署成功率，社区 PR 数量

## 第三部分：新用户引导向导设计（P0 核心）

### 设计理念

基于 SaaS 最佳实践：

- **3-5 步**，不超过 7 步（超过 20 步完成率下降 30-50%）
- **首次价值时间 < 2 分钟**
- **渐进式信息收集**：先做最重要的事（导入数据），后做次要的事（个人设置）
- **可跳过**但有引导回归

### 技术实现

- **组件位置：** `apps/web/src/app/[locale]/onboarding/` 目录

- **可重入：** Settings 页面提供"重新运行引导"选项
- **i18n：** 完整 EN/ZH 支持

## 第四部分：法律页面设计（P0 核心）

### 服务条款 (ToS) 关键条款

1. **服务描述** — DTax 提供加密税务计算工具，不构成税务建议
2. **用户责任** — 用户对数据准确性负责，DTax 不验证交易数据
3. **免责声明** — DTax 计算仅供参考，用户应咨询专业税务顾问
4. **知识产权** — AGPL-3.0 核心开源，Pro/CPA 功能为商业许可
5. **账户终止** — DTax 保留终止违规账户的权利
6. **数据处理** — 引用隐私政策
7. **争议解决** — 仲裁条款
8. **SaaS 计费** — 按税年计费，不自动续费，退款政策

### 隐私政策关键条款

1. **收集的数据** — 邮箱、加密交易记录（用户上传）、使用日志
2. **不收集的数据** — 私钥、钱包密码、社交账号
3. **数据用途** — 仅用于税务计算，不用于广告、不出售给第三方
4. **数据存储** — 加密存储，用户可随时删除账户和所有数据
5. **自托管说明** — 自托管版本数据完全由用户控制
6. **第三方服务** — CoinGecko API（仅价格数据，无用户信息）
7. **Cookie 政策** — 仅功能性 cookie（JWT token），无追踪 cookie
8. **GDPR/CCPA 合规** — 数据导出权、删除权、更正权

### 税务免责声明

放在以下位置（加粗/高亮）：

- ToS 中独立章节
- 每个税务报告页面底部
- 首次计算税务时的弹窗确认
- Landing Page 定价区域下方

## 第五部分：静态页面架构

### 技术方案

所有新页面在现有 Next.js App Router 内实现，不引入新框架：

```
apps/web/src/app/[locale]/
├── page.tsx              ← 现有 (Landing/Dashboard)
├── auth/page.tsx         ← 现有
├── onboarding/page.tsx   ← 新：引导向导
├── pricing/page.tsx      ← 新：独立定价页
├── features/page.tsx     ← 新：功能详情页
├── security/page.tsx     ← 新：安全说明
├── exchanges/page.tsx    ← 新：交易所集成
├── legal/
│   ├── terms/page.tsx    ← 新：服务条款
│   ├── privacy/page.tsx  ← 新：隐私政策
│   └── disclaimer/page.tsx ← 新：税务免责
├── docs/
│   ├── page.tsx          ← 新：文档首页
│   ├── self-host/page.tsx ← 新：自托管指南
│   └── changelog/page.tsx ← 新：更新日志
├── for-cpas/page.tsx     ← 新：CPA 介绍页
└── faq/page.tsx          ← 新：常见问题
```

### 公共页面访问控制

这些页面无需登录即可访问，AuthGuard 需要更新：

```typescript
// auth-guard.tsx — 公共路径列表
const PUBLIC_PATHS = [
  "/auth",
  "/pricing",
  "/features",
  "/security",
  "/exchanges",
  "/legal",
  "/docs",
  "/for-cpas",
  "/faq",
  "/onboarding",
];

const segments = pathname.split("/").filter(Boolean);
const subPath = "/" + (segments[1] || "");
if (segments.length <= 1 || PUBLIC_PATHS.some(p => subPath.startsWith(p))) {
  return <>{children}</>;
}
```

### Nav 策略

- **未登录：** Logo + 功能 / 定价 / 文档 / Sign In
- **已登录：** 现有完整导航（Dashboard, Transactions 等）
